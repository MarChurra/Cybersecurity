## GAP Analysis
- Where you are compared where you want to be, the gap between them
- It amy require extensive research

## Choosing the framework
- Work towards a known baseline
  - An internal set of goals
  - Can use formal standards
- Determine the end goal
  - NIST Special Publication800-171 Revision2 
  - ISO IEC 27001
- Get a baseline of employees
  - Formal experience
  - Current training
  - Knowledge of security policies and procedures
- Examine the current processes
  - Reasearch existing IT systems and policies

## Compare and contrast
- The comparison
  - Evaluate existing systems
- Identify the weakness
  - Along witht he most effective processes
- A detailed analysis
  - Examine broad security cateogries
  - Break those into smaller segments
- The final comparison
  - Detailed baseline objectives
  - A clear view of the current state
- Need a path to get from the current security to the goal
  - Requires time, money, and changes to control
- Time to create the gap analysis report
  - A forma description of the current state
  - Recommendations for meeting the baseline
